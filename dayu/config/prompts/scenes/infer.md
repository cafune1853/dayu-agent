# 公司业务类型与关键约束判断执行契约

## 任务目标
- 先判断“这家公司主要是什么生意”，再判断“哪些关键约束会显著改变后续章节写作的判断路径”。
- 这一步的结果只服务后续章节写作裁剪，不直接生成任何章节正文。
- 下游是按固定模板逐章写作；因此这里的目标不是做行业分类展示，而是先为模板写作确定“判断分岔点”。
- `business_model_tags` 应让同一章节在面对不同行业/不同商业模式公司时，走出明显不同的判断路径。
- `constraint_tags` 应让同一行业里的不同公司，围绕各自特殊结构变量写出不同的判断重点。

## 做什么
- 若显式输入不足以稳定归因，可按既有财报工具工作流最小检索；优先读取最新年报。
- 只从给定候选中选出最能代表“主业务类型”和“关键约束”的结果，并给出极简说明。

## 不做什么
- 不自由发散创造新标签。
- 不写章节正文、不做行业长分析、不输出买卖建议。
- 不为了凑数量而多选标签；不确定时宁可少选。

## 执行方式
- 需要工具时直接调用工具；若要读财报，先 `list_documents`，再按需要读取最新年报相关内容。
- 最终只输出一个 JSON 对象，不写解释性前缀或 Markdown 包装。
