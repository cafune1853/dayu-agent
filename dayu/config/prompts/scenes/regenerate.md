# 整章重建执行契约

## 任务目标
- 仅在结构性失败时，基于骨架与修复合同整章重建正文。

## 做什么
- 优先修复结构问题，并在既有章节目标内重建完整正文。
- 关键断言写回正文时，同步重建对应的证据锚点与“### 证据与出处”。

## 不做什么
- 不补占位符；遇到无法补证的事实，按占位符规范保留缺口，不要为补齐缺口而重写相邻章节。
- 不做局部 patch 或针对单条问题的修修补补；本轮只输出完整章节正文，不输出 patch。
- 不输出审计判断、违规分类或证据裁决 JSON；只负责写出合规正文与“证据与出处”。

## 执行方式
- 需要工具时直接调用工具；准备交付正文时，直接输出最终 Markdown 代码块，不写任何起手说明、过程复述或过渡句。
- 关键数字、比例、排名或具体断言写入正文时，必须同步选择最能直接支撑该句的具体证据锚点；若一条证据不足以稳妥支撑一句中的全部信息，就拆句、删弱或改写为更客观的表述。
- 若正文 claim 来自 `read_section` 或其它 section-path 定位工具，证据行必须引用实际承载该句的最窄 section；若 claim 只在父级 heading 中成立，不得用相邻子节或更窄但不承载该句的路径代替。
- 若某个条件项依赖 `get_financial_statement` / `query_xbrl_facts` 才能成立，必须先选定 statement/xbrl 锚点再写正文；做不到就不要写该句。
- `证据与出处` 只列正文实际使用过的来源；每行只写来源识别与定位信息，不夹带摘要、分析或结论。
