写作目标：
- 章节为“投资要点概览”。
- 把传入的前文章节结构化输入压缩成一页买方初筛封面：先让读者一眼知道这是什么生意、这是一家怎样的公司、现在该做什么，再快速交代为什么当前是这个动作、这家公司当前经营与财务处在什么状态、最该先抓住哪个变量、最大卡点是什么，以及下一步最小验证问题是什么。
- 这不是摘要汇总，也不是章节导读；你的职责是把前文章节输入收成最低认知负担、但信息密度足够高的首屏判断。
- 禁止引入当前输入之外的新事实、新数字、新来源。
- 预设读者是第一次接触或第一次系统性了解这家公司；不要假设读者已经知道公司背景、行业术语或历史争议。

输出要求：
- 仅输出 1 个 Markdown 代码块（且只输出这一块）。
  - 代码块内禁止用数字编号（不要出现“1.”“（1）”等）编排格式，可用无序列表与小标题。
  - 代码块内禁止用粗体、斜体等编排格式。
  - 不输出任何 HTML 注释内容（<!-- ... -->）。
- 仅输出该章节正文，禁止输出其它解释性文字。
- 不输出 “### 证据与出处”。
- bullet格式要求：
  - bullet下另起一行输出回答：第一行bullet，另起一行紧接着输出回答正文，二者之间不留空行。
  - 不得把bullet改写成非bullet。
  - 不得在bullet行直接追加回答内容、解释文字或额外标点；禁止写成 `- 标签：回答`、`- 标签 回答` 或 `- 标签：`。
  - 相邻 bullet 之间保留空行；不要把多个字段合并进同一 bullet，也不要把单个字段再拆成内层 bullet，除非下方章节结构已明确要求。
- 正文默认写成封面页三层结构：先用一个很短的“一眼看懂”区块给出“这是什么生意”“公司简介”和“现在该做什么”，再回答“为什么是现在这个动作”，最后回答“下一步怎么验证”。
- 输出标签只使用下方章节结构中面向读者的最终文案；不要把默认条数、阈值、bullet 约束等写作指示抄进正文。
- 整体语气要像写给买方同事、且对方第一次接触这家公司的封面导语，不要写成字段回填卡片、机器人摘要或模板腔很重的“标准答案”。

内容边界（强制）：
- 只能复用下方前文章节结构化输入中已出现的事实和数字；可以在不改变事实边界的前提下压缩、重组或改写表述，不得新增任何未在当前输入出现的信息。
- 本轮允许取材的来源边界已在下方给出；只能在该边界内取材。
- 若下方写明不允许新增事实，发现事实冲突或信息缺口必须按系统占位符统一格式（【占位符】+ 缺口/需要/已检索范围/下一步）闭环，不得补写新事实。
- 若下方写明不允许新增来源，不得引入当前输入中未出现过的新来源条目。
- 不把输出写成按章复述、信息罗列式摘要或“公司介绍 / 产品介绍 / 财务概览 / 股东回报”的并列分栏。
- 不输出“证据与出处”小节；第0章是封面页，不承载证据溯源。
- 先回答当前研究动作，再压缩出主判断链、主变量、主卡点和最小验证问题；不要先铺材料，再迟迟不落动作。
- `公司简介` 只保留帮助第一次接触这家公司的人快速建立画像的最必要背景；不要写历史沿革、愿景口号或多段宣传式介绍。
- `当前经营与财务处在什么状态` 只保留最能解释当前动作的经营、盈利、现金流或资产负债状态；不要把它写成迷你财报摘要，也不要完全不写经营/财务信息。
- 默认只保留 1 条主判断链、1 个主卡点、1 个最关键验证问题和 2 个阈值事件；只有缺一不可时才扩写。
- 若“先抓哪个变量”和“最大卡点”本质是同一件事，变量写成需要持续观察的观测点，卡点写成一旦验证失败会如何推翻当前动作，避免重复复述。
- `当前最大卡点` 要写当前最阻止升级或维持当前动作的现实阻碍；不要写成“如果未来变好，当前动作可能过于保守”这类反事实句。
- `下一步先验证什么` 默认只写 1 个最关键问题；不要把两个问题并列塞进同一项，除非两个问题彼此独立且任何一个单独成立都会改变当前动作。
